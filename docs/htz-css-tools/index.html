htz-css-tools

A set of functions and mixins for consistent style authoring in Haaretz apps

Table of Contents

  • Installation
  • Usage

Installation

$ yarn add @haaretz/htz-css-tools

or 

$ yarn install --save @haaretz/htz-css-tools

Usage

See the documentation

Create a function that returns the value of named-colors based on a preconfigured theme.

createColorGetter(colorPalette: ColorPalette): ColorGetter
Parameters

Returns

ColorGetter

packages/libs/htz-css-tools/src/color/createColorGetter.js

Get a color from a predefined color palette

ColorGetter

Type: function (color: string, variant: (false | string), altColorPalette: ColorPalette): string

Parameters
color (string) A named-color to get the value of from the palette
variant ((false | string) = false) A named variant of color to get from the palette. When color has variants, but variant is false , the base variant will be used by default.
altColorPalette (ColorPalette?) an alternative palette in place of the default one to get a color from. Can be Useful when using external elements the should be styled differently from branded elements.
Returns
string: A CSS color value

ColorPalette

packages/libs/htz-css-tools/src/color/createColorGetter.js

A themed color palette

ColorPalette

Type: Object

Properties
colorName ((string | Array<string> | ColorPalette)?) : The value of a named-color, a reference to another named-color (or color variant), or color-variant definitions.
  • A string will be interpreted as a reference to another named-color, or the straight up value of the named-color.
  • An Array of strings can be used to pass references to a variant of a named-color. The item in position 0 of the array points to the referenced named-color, and the optional item in position 1 points to a named-color's variant name.
  • A nested ColorPalette object is used to define variants of a color.

    createMqFunc

    packages/libs/htz-css-tools/src/mq/mq.js

    Return a config-based media-query function.

    createMqFunc(breakpoints: BpsConfig?): MqFunc
    Parameters
    breakpoints (BpsConfig? = defaultBps)
    A configuration object containing settings for named breakpoints and their values
    Name Description
    breakpoints.widths WidthBpsConfig (default { s:600, m:768, l:1024, xl:1280, }) An object containing named width-breakpoints as keys, and a number representing their width in pixels as values. Used when evaluating the from and until arguments in media query functions.
    breakpoints.misc MiscBpsConfig (default { landscape:'(orientation: landscape)', partrait:'(orientation: portrait)', hidpi:'(min-resolution: 1.5dppx)', }) An object containing named breakpoints of commonly used miscellaneous media-query features as keys, and their associated query features as values.
    Returns
    MqFunc: A function that generates media-query strings based on the breakpoints config object.

    MqFunc

    packages/libs/htz-css-tools/src/mq/mq.js

    A function that intelligently returns a media-query scoped css-in-js object

    MqFunc

    Type: function (MqOptions, styles: Object): Object

    Parameters
    mqOptions (MqOptions)
    Name Description
    mqOptions.from (string | number)? The min-width argument (inclusive).
    A number will be interpreted as a value in pixels (and converted to ems).
    string s can either be a named breakpoint or number-strings in css length units, which Will not be converted.
    mqOptions.until (string | number)? The max-width argument (exclusive).
    A 'number' will be interpreted as a value in pixels (and converted to ems).
    Strings can either be a named breakpoint or number-strings in css length units, which Will not be converted.
    mqOptions.misc string? Miscellaneous media feature queries. Either a named breakpoint, or a feature string, e.g., (orientation: landscape) .
    mqOptions.type string? A media type, e.g., only screen , print , etc.
    styles (Object) A css-in-js style object
    Returns
    Object: A media-query scoped css-in-js object
    Example
    const mq = createMqFunc();

// Styles as object
const style = {
  ...mq({from: 's', until: 'l'}, { color: 'red', }),
}

// Output
const style = {
  '@media (min-width: 37.5em) and (max-width: 64em)': {
    color: 'red',
  }
}

    TypeConf

    packages/libs/htz-css-tools/src/typography/confTypes.js

    Typographic Scale Configuration
    A configuration object Containing named breakpoints as keys, and TypeScaleOpts objects as values for each such breakpoint.
    Must contain at least a default key, holding the default typographic scale options.

    Breakpoints for which TypeScaleOpts aren't explicitly set, will inherit the options of the breakpoint preceding them.
    With the exception of default, which is mandatory, breakpoint names must correspond with those defined in the breakpoint configuration. Larger breakpoints override smaller ones.

    TypeConf

    Type: {default: TypeScaleOpts}

    Properties
    default (TypeScaleOpts)
    Example
    const typeConf = {
  default: {
    base: 16,
    minPadding: 2,
    ratio: 2,
    rhythmUnit: 6,
    steps: 5,
  },
  l: {
    base: 18,
    minPadding: 2,
    ratio: 2,
    rhythmUnit: 7,
    steps: 5,
  },
};

    TypeScaleOpts

    packages/libs/htz-css-tools/src/typography/confTypes.js

    Base configuration options for setting up harmonious typographic scales.

    TypeScaleOpts

    Type: {base: number, steps: number, ratio: number, rhythmUnit: number, minPadding: number}

    Properties
    base (number) : The base font-size , usually set on the body element.
    steps (number) : The number of steps in between intervals of the scale.

    If x is a size in the scale, it will be incremented to x * ratio after specified number of steps.

    ratio (number) : The typographic scale ratio. The multiplier by which the scaling property of the scale is determined.

    If x is a size in the typographic scale, than x * ratio will also be size in the ratio, and will occur after n steps, as defined in steps.

    rhythmUnit (number) : The basic spacing unit used throughout to construct consistent rem -based vertical rhythm systems by setting the font size property on the html element.
    minPadding (number) : The minimal amount of padding (in pixels) each line should have below and above the text. A font-size 's line-height must, at minimum, conform to: (lineHeight - fontSize) / 2 >= minPadding .
    base (number)
    steps (number)
    ratio (number)
    rhythmUnit (number)
    minPadding (number)
    Example
    {
  base: 18,
  minPadding: 2,
  ratio: 2,
  rhythmUnit: 7,
  steps: 5,
},

    Typesetter

    packages/libs/htz-css-tools/src/typography/createTypesetter.js

    A function returning a CSS-in-JS object of typographic styles conforming to a global predefined typographic scale and vertical rhythm. Created by createTypesetter()

    Typesetter

    Type: function (step: number, options: TypesetterOpts): (TypographicStyles | {})

    Parameters
    step (number) The step in the scale to get. The typographic scale is 0 based, so 0 base font-size (the size that is set on <body> )
    options (TypesetterOpts?)
    Name Description
    options.lines number? The number of rhythm-lines the line-height should span.
    options.fromBp string? A named breakpoint, used for a min-width query
    options.untilBp string? A named breakpoint, used for a max-width query
    options.pxFallback boolean? Should rem values have px fallback
    Returns
    TypographicStyles: A CSS-in-JS object of typographic styles conforming to a global predefined typographic scale and vertical rhythm.
    Example
    const typesetter = createTypesetter();

// Assuming the `m` breakpoint is `600px`,
// the default rhythm unit is `6px` and
// the default base `font-size` is `16px`:
typesetter(1, {until: 'm'}); // => {
                             // '@media (max-width; 37.5em)' {
                             //     fontSize: '2.6666666667rem',
                             //     lineHeight: '1.5em',
                             //   },
                             // };

    Border Mixins

    Border mixins that will add the right amount of padding to elements so that borders don't break the vertical rhythm

    border

    packages/libs/htz-css-tools/src/border/index.js

    Set border related styles on all sides of an element, in a manner that doesn't mess up the vertical rhythm

    border(optsOrWidth: (BorderOptions | number | string), lines: number, style: BorderStyle, color: string): BorderRuleset
    Parameters
    optsOrWidth ((BorderOptions | number | string)) An options object, or the width of the border. When an object, must be the only argument passed
    • `number` values will be interpreted as being in pixels
    • `string` values will be used as is
    lines (number) The number of vertical rhythm lines the border + additional padding should occupy. Mandatory when if not passing an object as the first arg.
    Setting lines to 0 , will make the element relatively positioned, and create a pseudo element in the height specified in width , stuck to the relevant edge(s)
    style (BorderStyle) The border-style
    color (string) The border-color
    Returns
    BorderRuleset: css-in-js object with border related styles, and the correct padding ensuring the vertical rhythm is kept in tact
    Example
    border('2px', 1, 'solid', 'red');
border({ width: '2px', lines: 1, style: 'solid', color: 'red' });

// Both return
{
  borderBottomColor: 'red',
  borderBottomStyle: 'solid',
  borderBottomWidth: '2px',
  borderInlineEndColor: 'red',
  borderInlineEndStyle: 'solid',
  borderInlineEndWidth: '2px',
  borderInlineStartColor: 'red',
  borderInlineStartStyle: 'solid',
  borderInlineStartWidth: '2px',
  borderTopColor: 'red',
  borderTopStyle: 'solid',
  borderTopWidth: '2px',
  paddingBottom: 'calc(1rem - 2px)',
  paddingTop: 'calc(1rem - 2px)',
}

// With zero lines of vertical-rhythm
border('2px', 0, 'solid', 'red');

// Retruns
{
  borderInlineEndColor: 'red',
  borderInlineEndStyle: 'solid',
  borderInlineEndWidth: '2px',
  borderInlineStartColor: 'red',
  borderInlineStartStyle: 'solid',
  borderInlineStartWidth: '2px',
  position: 'relative',

  ':before': {
    backgroundColor: 'red'
    content: '',
    height: '2px',
    position: 'absolute',
    top: '0',
    width: '100%',
  }

  ':after': {
    backgroundColor: 'red'
    bottom: '0',
    content: '',
    height: '2px',
    position: 'absolute',
    width: '100%',
  }
}

    GetTypePropsOpts

    packages/libs/htz-css-tools/src/typography/getTypeProps.js

    Options object for the getTypeProps function

    GetTypePropsOpts

    Type: {step: number, base: number?, rhythmUnit: number?, minPadding: number?, ratio: number?, stepsPerInterval: number?}

    Properties
    step (number) : The step in the scale to get. The typographic scale is 0 based, so 0 here will return the number set in base
    base (number?) : The font-size at the base of the scale
    rhythmUnit (number?) : The basic unit used to set the vertical rhythm to which the line-height should conform. The typographic scale ratio. The multiplier by which the scaling property of the scale is determined.

    If x is a size in the typographic scale, than x * ratio will also be size in the ratio, and will occur after n steps, as defined in stepsPerInterval.

    minPadding (number?) : The minimal amount of padding (in pixels) each line should have below and above the text. A font-size 's line-height must, at minimum, conform to: (lineHeight - fontSize) / 2 >= minPadding .
    ratio (number?)
    stepsPerInterval (number?) : The number of steps in between intervals of the scale.

    If x is a size in the scale, it will be incremented to x * ratio after the specified number of steps.

    PxToUnitConverter

    packages/libs/htz-css-tools/src/units/pxTo.js

    A function that converts pixel values to target unit designated by the factory function.

    PxToUnitConverter

    Type: function (pxVal: (number | Array<number>), context: number): string

    Parameters
    pxVal ((number | Array<number>)) The number, in pixels to convert to the specified unit.
    context (number) The number, in pixels to be used as the base for converting pxVal to the target unit.
    Returns
    string: A converted number-string in the target unit.
    Example
    // Create a `px` to `pt` converter
const pxToPtConverter = pxTo('pt');

// pt is always (px * 0.75 )
const pxToPt = (pxValue) => pxToPtConverter(pxValue, 0.75);
